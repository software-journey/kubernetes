# Episode 13: The Night Shift Nobody Talks About 🌙

Jobs and CronJobs are for work that should run to completion (once, or on a schedule)—the “night shift crew” in a version-controlled, observable way.

## 🎯 What You’ll Learn

- The difference between **Jobs** (run once) and **CronJobs** (run on a schedule)
- Why Kubernetes is safer than “a mysterious crontab on one server”
- How retries, history, and logs make batch work tractable
